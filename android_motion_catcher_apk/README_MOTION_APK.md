# Motion Catcher APK

这是一个独立 Android APK 工程，目录为 `android_motion_catcher_apk`，没有修改现有 `android_catcher_apk` 或 `android_api_catcher_apk`。

它的控制流程和旧版大致一致：

- `MediaProjection` 持续截屏。
- `AccessibilityService` 在你手动启用后执行滑动手势。
- 悬浮条提供 `抓捕`、`暂停`、`停止`、`标定`。
- 准星和丢球键仍然用固定屏幕坐标标定。

不同点是：本版本不使用 YOLO、不需要 `model.tflite`、不需要训练数据。它会观察连续画面，把几秒内从一个位置大幅移动到另一个位置的紧凑运动像素块判定为精灵。

从 `0.3.0` 开始，抓捕不是固定频率盲滑：

- 有精灵：从准星位置做小步动态滑动，把准星往精灵中心对齐。
- 对齐：精灵进入丢球半径后，短按丢球键。
- 没精灵：保持当前画面，超过默认 `5 秒` 后再随机向上/下/左/右探索一次。
- 探索后：清空一次运动历史，等新画面稳定后重新识别，避免整屏移动导致检测器卡在“全局变化”状态。

从 `0.4.0` 开始，识别增加了“暗色活动目标”启发式：除了帧差轨迹，还会寻找局部明显更暗、偏蓝紫、大小紧凑的候选，并匹配它从上一帧位置到当前帧位置的位移。它专门用于恶魔狼这类深色目标在蓝色雪地背景中移动但帧差轨迹不稳定的场景。

## 运动识别思路

`MotionSpriteDetector` 每隔一段时间采样屏幕网格：

1. 对采样颜色做亮度归一化，减少整体光影变化造成的误判。
2. 用相邻帧差异找出运动像素块，并过滤过大的全屏变化、转场、强光影闪烁。
3. 把紧凑的运动块连成 blob，再在几秒历史窗口内跟踪轨迹。
4. 如果某条轨迹移动距离超过阈值、持续帧数足够、形状不太像全屏环境变化，就输出为 `sprite`。
5. 精灵转身导致颜色变了也不会立刻丢弃轨迹，因为相邻帧连接主要看位置连续性，颜色只作为弱约束。

## 构建

本目录提供不依赖 Gradle wrapper 的构建脚本。脚本会优先使用本目录 `.android-sdk`，没有的话会复用仓库中已有 APK 工程的 Android SDK 工具链。

```powershell
cd D:\rock\android_motion_catcher_apk
powershell -ExecutionPolicy Bypass -File .\build_apk.ps1
```

构建成功后 APK 位于：

```text
D:\rock\android_motion_catcher_apk\app\build\outputs\apk\debug\app-debug.apk
```

## 安装

如果继续使用仓库已有 Android SDK 的 adb：

```powershell
D:\rock\android_catcher_apk\.android-sdk\platform-tools\adb.exe install -r D:\rock\android_motion_catcher_apk\app\build\outputs\apk\debug\app-debug.apk
```

安装后应用名是 `Motion Catcher`，包名是 `com.example.rockcatchermotion`，可以和旧版 `Rock Catcher` 同时存在。

从 `0.2.0` 开始，本工程固定使用根目录的 `debug.keystore` 签名。保持包名不变并使用 `adb install -r` 覆盖安装时，Android 会按同一个 app 升级处理，应用数据和已授予的权限会尽量保留。当前版本是 `0.4.0`。

## 使用流程

1. 打开 `Motion Catcher`。
2. 点 `打开无障碍设置`，启用 `Motion Catcher Gestures`。
3. 回到应用，点 `启动悬浮条`，同意系统截屏授权。
4. 进入游戏。
5. 如果还没标定坐标，点悬浮条 `标定`，屏幕上会出现 `准星` 和 `丢球键` 两个可拖拽符号。
6. 把两个符号拖到真实位置，点上方 `完成`。坐标会立刻保存，抓捕逻辑会直接读取新坐标。
7. 再用悬浮条：
   - `抓捕`：开始识别、动态瞄准、对齐后点击丢球键。
   - `暂停`：继续识别，但不控制手机。
   - `停止`：结束截屏和接管服务。
   - `标定`：在当前游戏画面上拖动准星和丢球键坐标。
   - 长按 `标定`：保存当前截图，方便离线排查。

旧版 `录制` 的作用是保存当前截图，用来离线标定或排查识别。新版把悬浮条按钮升级成现场标定；长按同一个按钮仍然保留截图录制，主界面里的 `标定准星 / 丢球键` 也可以继续用截图离线标定。

## 常用调参

- 主界面默认只显示常用设置；点 `显示高级参数` 可以看到全部识别和探索参数。
- `准星接近精灵时的丢球半径 px`：默认 `42`。太早扔就调小，迟迟不扔就调大。
- `采样步长 px`：默认 `14`。识别不稳可调小，例如 `10`；手机发热或卡顿可调大。
- `运动阈值`：默认 `18`。误报多就调高；识别不到就调低。
- `全屏变化过滤`：默认 `0.65`。强光影或转场误报多就调低；镜头运动时经常不识别就调高。
- `观察历史 ms`：默认 `3500`。精灵移动慢就调大。
- `几秒内最小移动距离 px`：默认 `130`。误报多调高；精灵小幅移动也要抓就调低。
- `相邻帧轨迹连接距离 px`：默认 `240`。精灵移动很快或帧率低就调高。
- `最低轨迹分`：默认 `0.44`。误报多调高；漏检多调低。
- `忽略准星半径`、`忽略丢球键半径`：用于避免准星和按键自身动画被识别成精灵。
- `X/Y 方向`：新版默认 `Y=-1`，避免一直往下滑导致视角看脚底。如果方向仍然反了，把对应方向乘以 `-1`。
- `X/Y 灵敏度`、`最大滑动步长 px`：用来控制每次拖动丢球键的幅度。屏幕滑动太快时，优先降低 `最大滑动步长 px`。
- `手势时长 ms`：越大越慢，默认 `520`。
- `两次手势间隔 ms`：越大越稳，默认 `220`。
- `手势后画面稳定等待 ms`：刚滑完屏幕时先停一会儿，避免把镜头运动当成精灵运动，默认 `160`。
- `瞄准平滑`：越大越稳、越慢，默认 `0.55`。
- `无目标等待多久再探索 ms`：默认 `5000`，也就是没看到精灵时先看 5 秒，再随机探索。
- `探索滑动距离 px`：默认 `220`。找不到画面时探索太小就调大，镜头晃太大就调小。

## 注意

这个版本是本地视觉和手势原型，不包含绕过检测、防封、注入、改包、读内存或批量账号功能。运动像素方案依赖场景表现，第一次使用建议先用 `暂停` 状态观察主界面状态文本，再进入 `抓捕`。
